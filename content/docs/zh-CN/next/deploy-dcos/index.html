在 DC/OS 上部署 Pulsar

Tips
To enable all built-in Pulsar IO connectors in your Pulsar deploymente, we recommend you use apachepulsar/pulsar-all image instead of apachepulsar/pulsar image; the former has already bundled all built-in connectors.

DC/OS (the DataCenter Operating System) is a distributed operating system for deploying and managing applications and systems on Apache Mesos. DC/OS is an open-source tool created and maintained by Mesosphere.

Apache Pulsar 可作为Marathon Application Group，它以可管理的集合方式运行多个应用程序。

先决条件

You need to prepare your environment before running Pulsar on DC/OS.

  • DC/OS 版本 1.9 或更高

  • 一个至少有三个代理节点的DC/OS cluster

  • 已安装 DC/OS CLI tool

  • 来自 Pulsar GitHub repo 的PulsarGroups.json 配置文件。

    $ curl -O https://raw.githubusercontent.com/apache/pulsar/master/deployment/dcos/PulsarGroups.json

DC/OS 管理的 Mesos 集群中的每个节点必须至少有：

  • 4 个 CPU
  • 4 GB 内存
  • 60 GB 总磁盘

Alternatively, you can change the configuration in PulsarGroups.json accordingly to match your resources of the DC/OS cluster.

使用 DC/OS 命令界面部署 Pulsar

通过此命令在 DC/OS 上部署 Pulsar：

$ dcos marathon group add PulsarGroups.json

此命令在三组中部署 Docker 容器实例，构成了一个 Pulsar 集群：

  • 3 个 bookie（每个代理节点上有 1 个 bookie 和 1 个 bookie recovery实例）
  • 3 个 Pulsar broker（每个节点 1 个 broker 和 1 个 admin 实例）
  • 1 个 Prometheus 实例 和 1 个 Grafana 实例

When you run DC/OS, a ZooKeeper cluster will be running at master.mesos:2181, thus you do not have to install or start up ZooKeeper separately.

After executing the dcos command above, click the Services tab in the DC/OS GUI interface, which you can access at http://m1.dcos in this example. You should see several applications during the deployment.

BookKeeper 组

To monitor the status of the BookKeeper cluster deployment, click the bookkeeper group in the parent pulsar group.

At this point, the status of the 3 bookies are green, which means that the bookies have been deployed successfully and are running.

You can also click each bookie instance to get more detailed information, such as the bookie running log.

若要显示 ZooKeeper 中的 BookKeeper 信息，可以查看 http://m1.dcos/exhibitor。 在此例中，3 个 bookie 在 available 目录下。

The Pulsar broker group

Similar to the BookKeeper group above, click brokers to check the status of the Pulsar brokers.

You can also click each broker instance to get more detailed information, such as the broker running log.

Broker cluster information in ZooKeeper is also available through the web UI. 在此例中，可以看到 loadbalance 和 managed-ledgers 目录已经创建。

Monitor group

** monitory** 组由 Prometheus 和 Grafana 组成。

Prometheus

Click the instance of prom to get the endpoint of Prometheus, which is 192.168.65.121:9090 in this example.

If you click that endpoint, you can see the Prometheus dashboard. All the bookies and brokers are listed on http://192.168.65.121:9090/targets.

Grafana

Click grafana to get the endpoint for Grafana, which is 192.168.65.121:3000 in this example.

If you click that endpoint, you can access the Grafana dashboard.

在 DC/OS 上运行简单的 Pulsar 消费者和生产者

现在已经完全部署好了 Pulsar 集群，接下来就可以运行一个简单的消费者和生产者来展示 DC/OS 上 Pulsar 的运作。

下载并准备 Pulsar Java 教程

可以使用 git clone 下载 Pulsar Java tutorial 仓库。 This repo contains a simple Pulsar consumer and producer (you can find more information in the README file in this repo).

$ git clone https://github.com/streamlio/pulsar-java-tutorial

Change the SERVICE_URL from pulsar://localhost:6650 to pulsar://a1.dcos:6650 in both ConsumerTutorial.java file and ProducerTutorial.java file.

pulsar://a1.dcos:6650 端点用于 broker 服务。 通过从 DC/OS GUI 中获取每个 broker 实例的端点详细信息。 a1.dcos is a DC/OS client agent that runs a broker, and you can replace it with the client agent IP address.

Now, you can change the message number from 10 to 10000000 in the main method in ProducerTutorial.java file to produce more messages.

Then, you can compile the project code using the command below:

$ mvn clean package

运行消费者和生产者

执行此命令来运行消费者：

$ mvn exec:java -Dexec.mainClass="tutorial.ConsumerTutorial"

运行此命令来运行生产者：

$ mvn exec:java -Dexec.mainClass="tutorial.ProducerTutorial"

You see that the producer is producing messages and the consumer is consuming messages through the DC/OS GUI.

查看 Grafana metric 输出

While the producer and consumer are running, you can access the running metrics from Grafana.

卸载 Pulsar

You can shut down and uninstall the pulsar application from DC/OS at any time in one of the following two ways:

  1. Click the three dots at the right end of Pulsar group and choose Delete on the DC/OS GUI.

  2. Use the command below.

    $ dcos marathon group remove /pulsar
